HEXO 博客搭建（2）hexo框架配置简介

发表于2022-01-27|更新于2022-02-01|技术经验博客
|阅读量:

我们使用HEXO初始化博客完成后，可以看到整体的文件树结构如下，下面对重要的文件夹和文件逐项进行介绍：

node_modules

node_modules是HEXO框架的相关依赖包，即我们之前在博客文件夹中输入 npm run 时安装的各种包文件，目前使用的butterfly主题文件也已经以依赖包的形式安装到了node_modules当中，_congfig文件即为主题的配置文件，相关的页面设置均可以在里面进行修改和自定义，路径如下：

public

public 是HEXO 在发布到时编译出的浏览器可渲染文件（我是这么理解的）可以在里面看到自己写的文章被编译成了html格式的文件

scaffolds

scaffolds 是自己新建页面时的初始化标准，类似模板，可以参考官方文档进行自定义配置：

source

source文件夹为用户自己的资源文件夹，包括新建的页面、自己的图片等都应当优先放入此处

themes

themes是默认的主题配置文件夹，但由于我们的butterfly已经整合到了node的包里，所以不用进行操作

_config.yml文件

这一文件是对博客全局的基础性配置文件，包括作者、网站名字、副标题、邮箱、部署位置等都在这里。
